   (b) v0.1.1로 새로 낼지 사용자와 상의. 재빌드 후 **실제로 속도가 정상化됐는지, 멤버 목록도
   이번엔 성공하는지 라이브로 재확인이 꼭 필요하다** — 코드 리뷰만으로 끝난 게 아니라 이게
   이번 수정의 진짜 검증이다.

### 참고

- `acquire/collector.js`·`acquire/capture/*`는 CLAUDE.md 규칙대로 **여전히 손대지 않는 게
  기본값**이다. §M에서 멤버 목록의 15초 타임아웃이 의심되는 원인 중 하나로 지목됐지만, 근본
  원인(OneDrive)을 고치는 쪽으로 대응했고 이 파일 자체는 안 건드렸다 — 재검증 후에도 여전히
  타임아웃이 문제면 그때 이 값을 늘리는 걸 고려(가벼운 변경이라 이땐 collector.js를 건드려도
  괜찮음, 로직 자체가 아니라 상수값 조정이라 M1 라이브 검증을 무효화하지 않음).
- 실기동(Electron)이 필요하면 매번 **재수집 비용**(밴드 전체 재순회, 로그인/2FA 필요할 수
  있음)을 사용자에게 미리 알리고 진행 여부를 확인할 것.
- 실데이터가 이제 **세 곳**에 있다: 리포 내부 `data/raw/`(M3, gitignored), 사용자 Windows
  머신의 OneDrive `문서\밴드 성적 자동 채점기\`(M5 초기 검증용, 미완성 상태로 방치 — 마이그레이션
  안 함), 그리고 다음 세션 이후로는 `%APPDATA%\밴드 성적 자동 채점기\`(새 기본 위치)에 쌓인다.
- `score/`·`lib/` 쪽 로직은 M2(136건) + M3 실데이터 관통 + M4 디스퓨트 리허설 + M5 실데이터
  라이브 경로 + §L 과제점수(146건)로 깊게 검증된 상태. 추가 버그를 찾으러 다니기보다는 §M
  재검증과 §J 백로그의 차단 조건 해소 여부 확인이 우선이다.
