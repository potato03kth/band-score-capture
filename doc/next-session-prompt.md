## 지금 상태 — M5 Windows 배포 패키징 완료(2026-07-22)

`doc/PLAN.md` §K에 이번 세션 전체 기록이 있다. 요약:

- 목표: 교수님(비전문가, Windows 11)이 스스로 설치·실행할 수 있게 패키징. `electron-builder`로
  NSIS 설치 프로그램(무서명) 만들고, 사용자 데이터는 `내 문서\밴드 성적 자동 채점기\`로 고정,
  수집+채점을 하나의 Electron 진입점(3지선다 대화상자)으로 통합.
- PR #1(`feat/windows-packaging`)로 구현 → 병합 완료(main, 커밋 `19c5a8b`).
- **사용자가 실제 Windows 11에서 전체 라이브 경로를 직접 검증함**: 설치 → 첫 실행(설정 자동
  생성) → 설정 입력 → 재실행 → 3지선다 → "데이터 수집 후 채점" → 실제 밴드 로그인 성공 → 실제
  수집 → 부적합 데이터 게이트가 걸려 채점 거부, 이때 개발자용 stack trace 아니라 한국어 안내
  메시지가 정상적으로 뜸(`showFriendlyError` 경로 실증). 최종 "채점 완료" 성공 화면까지는 안
  갔지만(그건 `부적합_데이터_확인.xlsx`를 실제 데이터 보고 채워야 하는 사용자 판단 작업), 사용자가
  "여기까지로 패키징 검증은 충분하다"고 판단해 마무리.
- 부수적으로 빌드 중 겪은 electron-builder의 알려진 Windows 이슈(winCodeSign 압축 해제 시
  심볼릭 링크 권한 부족)와 해결책(Windows 개발자 모드 켜기)도 §K에 기록해둠 — 이후 이 프로젝트를
  다른 Windows 머신에서 빌드할 때도 똑같이 걸릴 수 있으니 재발굴하지 말고 §K 참고.
- `doc/professor-guide.md` 신규 작성(교수님 전달용 한국어 설치/실행 안내).

## 다음 세션 시작점

M5 코드 작업은 끝났다. 남은 건 **GitHub Release 업로드**(사용자 Windows 머신에 이미 만들어진
설치 파일을 릴리스에 첨부하는 것) 정도 — 이것도 사용자가 원할 때 진행하면 된다, 세션 시작 시
먼저 나서서 재촉하지 말 것.

그 외에는 `doc/PLAN.md` §J "배포 후 개선 백로그"(로스터 학번·수집완료여부 오탐·다중 밴드·감사
시트 제외사유 컬럼)가 여전히 남아있다.

**§J는 동결 상태다 — 세션 시작 시 차단 조건을 재확인하지 말 것.** "배포 후 점검"은 사용자가
"배포완료"를 선언하기 전까지는 손대지 않는다는 뜻이고, 학번 리스트도 나머지 밴드 권한도
사실상 영구히 오지 않을 것으로 간주한다. 다음 세션 시작 시:

1. 사용자가 먼저 "배포완료"라고 선언하지 않았다면 §J 항목은 전부 그대로 대기 — 묻지 말고
   바로 사용자와 다른 작업을 상의한다(GitHub Release, §J 밖의 작업, 문서 정리 등).
2. §I-6에 있던 "동명이인 실사례 확인"도 로스터 학번 입력에 종속돼 있어 §J와 함께 동결 중.

### 참고

- `acquire/collector.js`·`acquire/capture/*`는 CLAUDE.md 규칙대로 **여전히 손대지 않는 게
  기본값**이다. M5에서도 건드리지 않았다(`acquire/main.js`의 앱 라이프사이클/오케스트레이션만
  변경).
- 실기동(Electron)이 필요하면 매번 **재수집 비용**(밴드 전체 재순회, 로그인/2FA 필요할 수
  있음)을 사용자에게 미리 알리고 진행 여부를 확인할 것.
- 실제 라이브 테스트로 생긴 real capture 데이터가 이제 **두 곳**에 있다: 리포 내부
  `data/raw/`(M3, 두 밴드 관통 검증용, gitignored)와 사용자 Windows 머신의
  `내 문서\밴드 성적 자동 채점기\`(M5 패키징 검증용, 리포 밖이라 git과 완전히 무관). 후자에는
  아직 미해결 `부적합_데이터_확인.xlsx`가 남아있다 — 사용자가 실제로 그 밴드 성적을 확정하고
  싶어지면 그 파일을 채우는 게 다음 작업이 된다(도메인 판단 필요, 개발자가 대신 채우지 말 것).
- `score/`·`lib/` 쪽 로직은 M2(136건 단위테스트) + M3 실데이터 관통 + M4 디스퓨트 리허설 + M5
  실데이터 라이브 경로로 깊게 검증된 상태. 추가 버그를 찾으러 다니기보다는 §J 백로그의 차단
  조건 해소 여부 확인이 우선이다.
