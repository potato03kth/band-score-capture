## 확인 필요

M1(`acquire/`)은 실기동 검증까지 끝났다(`doc/m1-live-findings.md` §27-28 — 다시 손댈 때만

**같은 날 Phase 0~8 전부 구현·테스트·커밋 완료**됐다. `npm test`(=
`node --test 'score/*.test.js' 'lib/*.test.js'`)는 136건 전부 통과 상태다. 상세 이력은
`doc/PLAN.md` §H-3 각 phase 항목(전부 취소선 처리됨) 참고.

**이 세션에서 추가로 한 일(Phase 0~6 구현 이후 사후 검증 + Phase 8):**
1. 합성 데이터로 M2 파이프라인 전체를 실제로 여러 번 실행해보는 사후 검증에서 버그 2건 발견·수정:
   - `score/roster.js` 임시 학번(TEST####)이 재실행마다 재배정되던 버그.
   - `score/gaps.js` — 확인 엑셀이 이미 있으면 재수집으로 새로 드러난 결손이 게이트에 반영
     안 되던 버그(Phase 6 핵심 목적과 직결).
2. `input/3_멤버리스트_<밴드>.xlsx`/`readMemberListExport`가 실제 파이프라인에서 안 쓰이는
   죽은 코드임을 발견 → 사용자 확인(교수가 이미 "수업참여 학생 리스트"를 갖고 있어 원래부터
   불필요했음) 후 삭제.
3. Phase 8(`score_logic.xlsx`) 구현 — `lib/scoreLogic.js`(로딩/검증, 1_설정.xlsx와 달리
   미기입을 막지 않음) + `score/scorer.js` 파라미터화(일일활동상한·일일점수상한·댓글배수·
   게시글배수) + `score/index.js`의 과제글포함여부 연결. 구현 중 Phase 6과의 상호작용
   버그(`applyMemberCommentCorrections`가 커스텀 배수/일일점수상한일 때 기존 score를 잘못
   재계산하던 문제)를 추가로 발견해 같이 수정.
