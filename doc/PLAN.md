### 로스터 / 신원 매핑
- 로스터(학번↔실명) 원본은 **엑셀**, 지금 미제공 → **테스트는 합성(임의) 학번**으로 동작. 실사용 전 실제 로스터 수령.
- **밴드 멤버리스트(엑셀)** 포맷 확정: `이름 | 소개 | 리더여부 | 휴대폰 | 이메일 | 가입일`. → **user_no 없음·학번 없음.** 여기서 얻는 것: 실명 목록, **리더여부(교수·조교 식별 단서)**, 가입일. (※ `user_no`는 엑셀이 아니라 API `get_members_of_band`에서 확보 — D0-R R-4.)
- **매핑 정책 = 자동등록 우선**(닉네임=실명 자동 join). **동명이인 감지 시 사용자 예외처리 강제**(아래 워크플로).

### 동명이인 처리 워크플로 (신규 확정)
1. 채점 스크립트가 활동 파싱 후 **이름 충돌**(한 실명 ↔ 2+ `user_no`, 또는 로스터 동명 2+)을 감지.
2. `input/4_동명이인_매핑.xlsx` 생성. 열:
   `이름 | id(user_no) | 최근활동유형(게시글/댓글) | 최근활동내용 | 가입일 | 학번(빈칸·노란색)`
   - "학번" 열은 **비워두고 노란색 채움**(사용자가 채워야 함을 시각적으로 인지).
   - 최근활동 유형·내용(+가입일)은 사용자가 "이 `user_no`가 누구인지" 식별하는 근거.
3. 사용자가 학번을 채워 **저장**.
4. **재실행** 시 스크립트가 채워진 엑셀을 읽어 `user_no→학번` 확정 매핑. (미기입 잔존 시 다시 경고하고 중단)

### 사용자 입력 = 엑셀 전용 (교수 친화, 신규 확정)
> 실사용자(교수)는 프로그램에 약하다. **CLI 인자·대화상자·JSON 편집을 사용자에게 요구하지 않는다.**
> 모든 사용자 입력은 **설치 폴더의 `input/` 폴더 안 엑셀**로만 주고받는다. 동명이인 파일에서 쓰던
> **노란칸(=여기 채우세요) 템플릿 패턴을 모든 입력으로 일반화**한다.

- **역할 분리(중요)**:
  - **개발자/조교(1회 셋업)**: 정찰로 얻은 엔드포인트 패턴·pacing 등 기계값은 `config/config.jsonc`에.
    교수는 이 파일을 **절대 열지 않는다.**
  - **교수(매 실행)**: `input/` 폴더의 엑셀 노란칸만 채운다.
- **동작 흐름**:
  1. 프로그램이 `input/`에 필요한 입력 엑셀이 없으면 **노란칸 템플릿을 자동 생성**하고
     "노란 칸을 채우고 저장한 뒤 다시 실행하세요" 안내 후 **중단**.
  2. 교수가 엑셀 노란칸을 채워 저장 → 재실행 → 프로그램이 읽어 검증(형식·필수값). 오류 시 다시 안내·중단.
  3. 실행 중 **동명이인 감지 시** `input/`에 동명이인 매핑 엑셀을 **따로 생성**(앞서 정의한 그 파일) →
     교수 기입 → 재실행에서 소비.
- **`input/` 폴더 구성**(파일명 앞 번호로 순서 유도):
  | 파일 | 생성 주체 | 내용(노란칸=사용자 기입) |
  |------|-----------|--------------------------|
  | `1_설정.xlsx` | 프로그램 생성 | 측정 시작일·종료일, 실행모드(test/production), 대상 밴드 목록(이름·URL/ID) |
  | `2_로스터.xlsx` | 프로그램 생성 | 학번·실명 (교수가 이미 가진 "수업참여 학생 리스트"를 보고 채움) |
  | ~~`3_멤버리스트_<밴드>.xlsx`~~ | ~~교수가 밴드에서 내보내 투입~~ | **폐기(2026-07-21, D-1 확정 노트 참고)** — 리더여부는 API(`role`)로, 이름은 교수의 수업참여 학생 리스트로 이미 확보되어 불필요 |
  | ~~`4_동명이인_매핑.xlsx`~~ | ~~동명이인 감지 시 프로그램 생성~~ | **미착수 상태로 폐기** — 동명이인은 CSV `매핑상태`(`ambiguous`)로만 표시 |
- **F-5 해결**: 정식 사용 시 측정 시작/종료일은 `1_설정.xlsx` 노란칸으로 강제 입력받는다.
  `production` 모드에서 날짜 미기입/무효면 실행 거부(스테일 config 실행 원천 차단). **별도 프롬프트/인자 불필요.**
- 결과물(CSV, unmatched 리포트, 감사로그)은 사용자가 편집하지 않으므로 `out/`·`logs/`에 그대로 둔다.

### 계측 규칙 최종 확정 (2차 답변 반영)
- **인정 활동 = 게시글·댓글·대댓글만.** 공감(좋아요)·투표·조회는 제외 (R2 확정).
- **학생이 쓴 게시글/댓글/대댓글이면 대상이 누구든(학생 자신 글 포함) 무조건 +1 대상** (R1 확정: 학생↔학생 포함).
- **측정시간범위(측정기간 KST)를 벗어난 글은 제외** (규칙2 재확인).
- **유일한 예외(확정 ✓)**: **교수의 과제글(`[과제n]`)에 달린 댓글만 계측 제외.** 교수의 **비과제** 게시글 댓글은 계측(브리프 규칙6 유지).
  과제글엔 과제댓글만 달리므로 **그 글의 댓글 전부를 무시**(개별 머리말 검사 불필요).
- **과제글 판정(R3 확정) = 작성자 ∈ `professorUserNos`(교수 본인, =15222358) AND 머리말 `^\[과제\d+\]`.** 둘 다여야 과제글.
  학생의 `[과제n]` 글·교수의 머리말 없는 글은 과제글 아님.
  - 규칙7(교수·조교 소거)은 **소거집합 = 밴드별 `role!=member` ∪ `taUserNos`(=84880771)** 로 필터. → **과제 판정은 교수(`professorUserNos`) 한정, 소거는 리더+조교**로 역할 분리. (조교 role은 밴드마다 다르므로 `taUserNos` 명시 소거 — D0-R R-4.)
- **총점 상한(cap)은 점수체계상 상한이며 기간과 무관. 교수가 직접 설정**(R4 확정) → `1_설정.xlsx` 노란칸(기본 50). `점수 = min(활동일수, cap)`.
- **최초작성일·삭제(R5 확정)**: 채점은 `created_at`(최초작성일) 기준. **수집 시점에 이미 삭제된 글은 API에 없어 자동 미계측.**
  수집 후 발생한 삭제는 능동 추적하지 않음(비용 과다 — "대충" 허용) → 사실상 최초관측 레코드가 유지됨.

### 감사·출력 (U10·P4 반영)
- **P4 승인.** 단 CSV 컬럼명은 교수가 바로 이해하도록 **각 컬럼에 상세 설명**을 붙인다(A-3). CSV는 rule11대로 정본 유지하되,
  교수용으로 **설명·근거가 포함된 엑셀**(`out/결과_및_감사_<timestamp>.xlsx`)을 함께 산출.

### R-2. 취득 아키텍처 확정 (intercept-only)
- **데이터 도메인 `api-kr.band.us`**(배치 `bapi-kr.band.us`, 인증 `auth.band.us`). `www.band.us`는 SPA 셸.
- **인증 = 요청별 sjcl 서명**(`bauth`, `bandWebAuthInfo`). 쿠키만으로 재요청하면 `"권한이 없습니다"`.
  → **replay 불가. 앱이 보내는 요청의 응답을 가로채는 intercept가 유일 경로.** preload가 페이지 스크립트보다 **먼저** XHR/fetch 몽키패치(전송은 XHR 주 사용).
- 로그인: 밴드 자체계정/소셜. **captcha 없음, 2FA 있음(밴드앱 푸시/SMS)**. Chromium 통과 확인 → `persist:` 파티션 재사용.

### R-4. 역할·소거 (규칙4·5·7)

- 로스터 매핑: `get_members_of_band`(name↔user_no) ⨝ 로스터 엑셀(name↔학번). 동명이인은 `user_no`로 분리 + `4_동명이인_매핑.xlsx`. **가입일(`created_at`)** 을 동명이인 보조식별(NU4)에 사용.

### R-5. 🚨 완전성 급소 — 대댓글 별도 수집 (M1 스코프 확대)
- 실측(post/47): **UI "댓글 40" = 최상위 댓글 28 + 대댓글 12**(체크섬 일치). post-level `get_comments`는 **최상위만** 반환.
- 대댓글은 부모 댓글의 `comment_count`로만 표시되고 **응답 본문에 내용·작성자 없음** → **`comment_count>0`인 부모마다 nested 조회 필수.**
  안 하면 **그날 대댓글만 단 학생 누락(억울한 0점)** — 규칙3·R2(대댓글 활동 인정)와 직결.
- **`total` 필드는 대댓글 포함 총계 → 완료판정 기준으로 쓰지 말 것.** 완료판정 3단계: ①피드 소진 ②최상위 댓글 소진 ③대댓글 소진 → 밴드별 `collection_complete`.
- **영향**: 수집 호출 수 급증(글×댓글×대댓글). 저속 페이싱 하 시간 재산정, 세션 분할/야간 배치 고려(M1 일정).

### R-6. 미확정 (M1 초기에 확정)
- 대댓글 nested 조회 정확한 파라미터(`content_type:"comment"` + 부모 comment_id) 실캡처
- 세션 재로그인 주기(2FA 재요청) 실측 — 장시간 관측 필요
- 수정된 글의 `created_at` 유지 여부 / `get_members_of_band` 대량 멤버 페이지네이션 커서
- 교수 제공 멤버리스트 엑셀 ↔ `author.name` 최종 대조(NU2)

## B. 파라미터 배치 — 교수 입력(엑셀) vs 개발자(config)

파라미터를 **누가 만지는가**로 두 곳에 나눈다. 코드에 하드코딩 금지.

### B-1. 교수 입력 = `input/1_설정.xlsx` (노란칸, 각 열에 상세 설명)

시트 `설정` — `항목 | 값(노란칸) | 설명`:

| 항목 | 값(노란칸) | 설명(엑셀에 그대로 표기) |
|------|-----------|--------------------------|
| 측정 시작일 | (노랑) | 계측을 시작할 날짜. `YYYY-MM-DD` (KST 00:00부터). 예: 2026-06-28 |
| 측정 종료일 | (노랑) | 계측을 끝낼 날짜. `YYYY-MM-DD` (KST 23:59까지). 예: 2026-07-16 |
| 총점 상한 | (노랑, 기본 50) | 한 학생이 받을 수 있는 최대 점수. 기간과 무관하게 교수가 정함 |
| 실행 모드 | (노랑, test/production) | 정식 채점이면 `production`. 이때 날짜가 비면 실행 거부 |

시트 `대상밴드` — `밴드 이름(노랑) | 밴드 URL 또는 ID(노랑)` (2~7행 미리 노란칸으로 준비):

| 밴드 이름 | 밴드 URL 또는 ID |
|-----------|------------------|
| (노랑) 예: 3분반 | (노랑) 예: `https://www.band.us/band/103239777/post` 또는 `103239777` |

> URL을 넣든 숫자 ID를 넣든 프로그램이 `bandId`로 정규화(NU8).

## C. 정찰(recon) 계획 (브리프 (c))

정찰은 **첫 마일스톤이자 최대 리스크**다. 여기가 막히면 전체가 막힌다. 상세 체크리스트와
관측 기록 양식은 별도 파일 **`doc/recon-findings.md`(템플릿 생성 완료)** 에 있다. 요약:

1. 밴드 웹 로그인(조교 계정) → 대상 밴드 진입.
2. DevTools Network에서 **피드 스크롤 / 글 열기 / 댓글 더보기** 시 JSON을 반환하는 내부
   엔드포인트 URL 패턴 식별.
3. 그 응답에 `user_key` · `created_at` · `content` · **댓글 목록**이 실리는지 확인.
4. 게시글 목록 응답에 `user_key`/댓글이 없고 **상세·댓글 조회가 별도**인지 확인(U7).
5. 댓글 **페이지네이션("더보기")** 트리거 방식(커서/오프셋/버튼) 확인.
6. **created_at 포맷**(epoch ms? ISO? 타임존?) 확인(E-계열).
7. **user_key 안정성**(같은 사람이 다른 밴드/다른 글에서 같은 key인지) 확인(U2).
8. **교수·조교 user_key** 눈으로 식별 → 기록(U3).
9. Electron Chromium에서 네이버/밴드 로그인이 **captcha/2FA 없이** 통과되는지 확인(U8).
10. 산출: `recon-findings.md`에 관측 엔드포인트·응답 필드 채우기.

> **정찰 성공 판정**: 위 3·5·6이 모두 "예"이면 preload 인터셉트 경로가 성립. 하나라도
> "아니오"면 대안(CDP debugger / DOM 폴백)으로 분기하고 일정 재산정.

### D-5. 실패 모드와 방어책

| 실패 모드 | 방어책 |

## E. Unknown Unknowns 심화 (브리프가 명시 안 한 위험의 근거)

### E-1. 신원 매핑이 진짜 급소다 (U1·U2)
규칙8은 "닉네임=실명, 동명이인은 user_key로 분리"라고 했지만, **user_key(밴드) → 학번(로스터)**
연결 고리를 무엇으로 하는지는 규칙에 없다. 조인 키는 실명뿐인데, 실명이 겹치는 순간(=규칙8이
존재를 인정한 그 동명이인) 자동 매핑이 불가능해진다. 즉 규칙8은 문제를 "user_key로 나눠라"까지만
풀고, "나눈 user_key를 어느 학번에 붙이나"는 열어뒀다. 해결책 후보:
- (권장) **닉네임 규약 강제**: 학생 닉네임을 `실명(학번뒤4자리)`로 통일 → 결정적 매핑.
- **수동 매핑 큐**: 충돌 건만 사람이 1회 확정, 그 매핑을 `roster.csv`에 캐시.
- 둘 중 하나는 반드시 채택해야 한다. 안 하면 동명이인의 점수는 원리적으로 확정 불가.

### E-2. "봤다 ≠ 수집했다" (U6·U7)
무한스크롤/댓글 더보기를 덜 소진하면 활동이 조용히 누락되어 억울한 0점이 나온다. 방어는
"가장 오래된 관측 글의 created_at이 측정시작일보다 과거가 될 때까지" 수집하고 밴드별
`collection_complete`를 남기는 것. 또한 목록 응답에 댓글/`user_key`가 없으면 상세 조회가
글마다 필요해져(U7) 저속 정책상 수집 시간이 급증한다 — 정찰에서 반드시 확인.

### E-3. 시간 경계 (E-계열)
`created_at` 포맷(epoch ms vs ISO vs 서버 로컬)에 따라 KST 변환이 틀리면 하루가 밀려
경계일 활동이 통째로 오분류된다. 00:00/23:59 경계는 단위테스트로 고정한다.

### E-4. 규칙의 사각지대 (R1·R2)
규칙은 "교수 글"만 과제/비과제로 나눈다. **학생이 쓴 글, 학생 글에 달린 댓글, 대댓글, 공감,
투표**의 계측 여부가 정의되지 않았다. 이건 해석이 아니라 **정책 결정**이라 네 확인이 필요하다.

### E-5. 개인정보·운영 (U9·U10)
학번·실명은 PII다. 로컬 보관·`.gitignore`·파기시점 합의가 필요하고, 이의제기 대비 감사로그가
없으면 성적 근거를 방어할 수 없다. 채점 파이프라인은 점수→활동레코드→원천파일·수집시각까지
역추적 가능해야 한다.

### E-6. 6/28 시작·엑셀 왕복으로 새로 생긴 위험 (2026-07-16 답변 이후)

- **NU1. 백필 이전 삭제분은 복구 불가.** 6/28~최초수집 사이에 작성 후 삭제된 글/댓글은
  내부 API에서 사라져 관측 자체가 불가. R5("삭제된 글은 미계측")와 **방향은 일치**하나,
  삭제 시점이 우리 수집 전이면 "그날 활동 있었음" 여부도 알 수 없다는 한계는 남음.
  완화책: **가능한 빨리·자주 수집**해 노출창 최소화, 한계를 운영 문서에 명시.
- **NU2. 멤버리스트 "이름" ↔ API 닉네임 일치 가정.** 자동매핑은 둘이 같은 문자열이라는 가정에
  의존. 밴드별 표시명이 다르거나 공백·특수문자 차이가 있으면 매핑 실패 → 정찰/초회 실행에서
  대조 필요. (정규화: trim/공백정리 후 비교)
- **NU3. 사용자가 편집·저장한 엑셀의 견고성.** 사용자가 `동명이인_매핑.xlsx`를 열어 학번을 채우다
  헤더를 바꾸거나 `.csv`로 저장하는 등 파손 가능. 되읽기 시 **헤더·필수열 검증**, 학번 형식(숫자/자릿수)
  검증, 미기입 잔존 시 명확히 중단하고 재요청.
- **NU4. 활동이 0인 동명이인 식별 곤란.** 충돌 user_key 중 측정기간 내 활동이 없는 사람은
  "최근활동내용"이 비어 식별 근거가 약함 → 매핑엑셀에 **가입일(멤버리스트)** 열을 함께 넣어 보조.

  재실행·dedup이 일관됨(예: user_key 정렬 순 `TEST0001…`). 랜덤 금지.
- **NU6. 과제 판정은 교수 한정(R3).** 이전의 "리더집합만으로 규칙4·5·7 처리" 단순화는 **철회**.

### E-7. 엑셀 전용 입력으로 새로 생긴 위험 (2026-07-16 2차 답변 이후)

- **NU7. 엑셀 날짜 셀 타입.** 엑셀은 날짜를 시리얼 숫자·문자열·지역 서식 등으로 저장 → 잘못 읽으면
